양자 법률적용 및 위계 필터링 엔진 개발 작업지시서

문서 ID: WO-QLH-2026-001 버전: 1.0 작성일: 2026-07-11 대상: SGP-Agent / 공공·비즈니스 법률 적용 엔진

1. 프로젝트 개요 (Project Overview)

본 프로젝트는 사용자가 처한 환경(국가, 특정 지자체, 소속 조직, 개인 업무 유형)에 따라 최상위 법인 헌법(LV 1)부터 최하위 매뉴얼(LV 8)까지의 법적 위계를 자동 분석하고, 두 주체(양자) 간 계약·분쟁·업무 프로세스에 적용되어야 할 적법한 준거 규정을 필터링·매핑하는 앱 엔진 기능을 개발하는 것을 목표로 합니다.

SGP-Agent 현재 구현과의 관계: 기존 sgp_quantum_legal_engine.dart는 형사 수사 도메인에서 형법·특별법·민사 2~3관점 양자 비교를 수행합니다. 본 작업지시서는 이를 8단계 법적 위계 전체로 확장하는 로드맵이며, Phase 1은 현장 형사(경찰) 도메인 LV 1~4 + 내부 지침(LV 7~8)에 우선 적용합니다.

2. 시스템 아키텍처 및 데이터 흐름

① 입력 파라미터 수집 → ② 위계 매핑 엔진 → ③ 양자 관계 검토 → ④ 충돌·상위법 우선 → ⑤ 트리 UI 출력
  1. 입력 파라미터 수집: 유저 프로필 및 상황 데이터 — 국가, 지자체 코드, 소속 기관/기업 ID, 실무 도메인 카테고리, 양자 행위자 A/B 프로필
  2. 법적 위계 매핑 엔진: 상위법 우선 원칙에 따라 DB 내 법령-규정 간 계층적 필터링 (Parent_ID 트리 순회)
  3. 양자 관계 검토: 행위자 A·B 간 교집합 규정 및 상충 규정 분석 — 충돌 시 상위법·광역 적용 범위 우선 배정
  4. 최종 결과 출력: 최상위 준거법 → 실무 매뉴얼까지 단계별 리스트, ⚠️ 충돌 경고, 액션 아이템

온디바이스(SGP-Agent) vs 서버형(공공·기업) 배치

구분SGP-Agent (현장)서버형 (Phase 2+)
LV 1~4 (국가법) 번들 JSON + 선택적 OTA (공식 채널 승인 후) 법제처 API 주 1회 동기화
LV 5~6 (조례·규칙) 지자체 코드별 캐시 공공데이터포털 + Cron
LV 7~8 (사규·매뉴얼) 단말 로컬 (수사관 입력·조직 배포 패키지) JWT 조직 ID 인증 API

3. 핵심 개발 데이터 세트 (Data Schema Guide)

앱에 적용할 법률 위계 데이터 구조는 아래 8단계 분류 체계를 엄격히 준수합니다.

위계 레벨 규정 층위 적용 범위 (Scope) 앱 내 필터링 조건 (Filtering Key)
LV 1 헌법 국가 전체 global_default — 모든 양자 관계의 최상위 전제
LV 2 법률 국가 전체 domain_tag — 소관 분야 (근로기준법, 형법, 개인정보보호법 등)
LV 3 대통령령 (시행령) 국가 전체 parent_law_id — LV 2 부모-자식 링크
LV 4 총리령·부령 (시행규칙) 국가 전체 parent_decree_id — 실무 서식·세부 기준
LV 5 조례 특정 지자체 local_gov_code — 예: 11(서울), 26(부산)
LV 6 규칙 (행정/지자체) 지자체/기관 local_gov_code + agency_id — 고시·훈령·지자체장 규칙
LV 7 내부 규정 (정관/사규) 특정 조직 org_id — 양자 계약 당사자 A/B 사규 매핑
LV 8 매뉴얼 (지침/가이드) 개인 실무 task_category — 직무·태스크 단위 End-User 가이드

4. 기능적 요구사항 (Functional Requirements)

  • 상위법 우선 원칙 알고리즘: LV 7·8이 LV 2·3과 충돌 시 ⚠️ 상위법 위반 경고 노출, 상위법 기준 가이드를 우선 표출 (HierarchyConflictResolver).
  • 양자(Cross-Filtering) 매칭 로직:
    • 예: 서울 소재 A기업 ↔ B기업 협력 실무
    • 국가법(LV 1~4) + 서울 조례/규칙(LV 5~6) + A/B 사규 교집합(LV 7) + 공통 매뉴얼(LV 8) 병합 파이프라인
    • 형사 도메인(SGP-Agent): 형법 ↔ 특별법 ↔ 민사 경로 = 기존 LegalPerspective 확장
  • 동적 트리 UI: 법률 클릭 → 시행령 → 시행규칙 → 조례 → 조직 규정 → 매뉴얼 Top-Down 아코디언/트리 뷰 (SgpLegalHierarchyTreeWidget — 신규).
Human-in-the-Loop: AI·규칙 엔진 출력은 준거법 후보 제안이며, 최종 법적 판단·체포·입건 결정 책임은 출동 수사관 본인에게 있습니다 (SGP-Agent 면책 바 유지).

5. 비기능적 요구사항 및 예외 처리

  • 데이터 동기화: LV 1~4 — 법제처 API·공공데이터포털 연동, 주 1회 Cron (서버형). SGP-Agent는 공식 OTA 채널 승인 전 kEnableRemoteOta=false 유지.
  • 보안 제어: LV 7~8 — 조직 ID 인증(JWT/단말 조직 패키지) 통과 세션만 조회. 수사자료 외부 공유 전 반출 확인 게이트 (sgp_legal_compliance.dart).
  • 통비법·정통법: STT는 버튼 조작식·세션 고지 게이트. 수사 데이터 서버 미전송(온디바이스).
  • 충돌 미해결: 상·하위법 판단 불가 시 [추후 보완] + 수사관 수기 확인 플래그.

6. API 명세 초안 (서버형 Phase 2)

POST /v1/quantum-legal/resolve
Content-Type: application/json
Authorization: Bearer <JWT>

Request:
{
  "actor_a": { "org_id": "ORG-A", "local_gov_code": "11", "task_category": "procurement" },
  "actor_b": { "org_id": "ORG-B", "local_gov_code": "11", "task_category": "procurement" },
  "situation": { "domain": "criminal", "raw_text": "...", "incident_type": "mutual_combat" },
  "options": { "include_manuals": true, "strict_hierarchy": true }
}

Response:
{
  "hierarchy_chain": [ { "level": 1, "node_id": "...", "title": "..." }, ... ],
  "perspectives": [ { "law": "...", "recommended": true, "precedent_guide": "..." } ],
  "conflicts": [ { "lower_node_id": "...", "upper_node_id": "...", "message": "상위법 위반 가능" } ],
  "action_guidance": "..."
}

온디바이스(SGP-Agent)는 동일 스키마를 dart 로컬 엔진으로 구현 — HTTP 불필요.
