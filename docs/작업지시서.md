## 3-1. Windows `device-id=windows` 오류 (2026-07-10 추가)

- **원인**: SGP-Agent는 Android 전용. Windows 데스크톱 프로젝트 없음.
- **조치**: Run 구성을 SM-S918N 고정
  - `.idea/runConfigurations/main_dart.xml` → `main.dart (SM-S918N)` + `--device-id=R3CW203HFGK`
  - `.vscode/launch.json` · `settings.json` → `deviceId: R3CW203HFGK`
  - `.idea/deploymentTargetSelector.xml` → 기기 고정
- **주의**: Android Studio 상단 기기 선택에서 **Windows / Chrome 말고 SM-S918N** 선택


1. **`Ticker` 타입 미발견 컴파일 오류**
   → `flutter/scheduler` import 누락이 원인. `Timer.periodic` 1초 방식으로 교체 완료.
2. **`sgp_native_bridge.dart` import 경로 오류**
   → `../native/...` 상대경로가 `lib/features/native/`를 가리킴.
   → `package:sgp_agent/native/sgp_native_bridge.dart` 패키지 경로로 통일.
3. **타임라인 `IntrinsicHeight`+`Expanded` 레이아웃 예외**
   → 고정 높이 연결선 방식으로 교체, 하단 패널 `Flexible` 처리.

## 4. Windows 보안 경고 (dart.bat / flutter.bat)

- **원인**: 서명 없는 `.bat` 실행 시 Windows 확인 창 + Run마다 cmd 경유
- **적용된 조치**:
  - `scripts\fix-windows-flutter-security.ps1` — Unblock + 신뢰구역 레지스트리 (`-ApplyRegistry`)
  - `scripts\add-flutter-trusted-zone.reg` / `add-lowrisk-bat.reg` (최후 수단)
  - `.idea\dart.xml`, `.vscode\settings.json` — Dart SDK를 `C:\src\flutter\bin\cache\dart-sdk`로 고정
  - `tools\flutter-direct.cmd` — dart.exe 직접 호출 (보안창 없음, 동작 확인됨)
  - `scripts\run-android.cmd` — 원클릭 단말 실행
- **남은 사용자 작업**: 보안창이 뜨면 「항상 확인」 해제 후 실행 (flutter.bat / dart.bat 각 1회)
